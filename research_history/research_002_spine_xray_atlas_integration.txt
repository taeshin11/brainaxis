================================================================================
EVALUATION — WHICH DATA TO USE
================================================================================

CANDIDATE A: CTSpine1K (0019_CTSpine1K_Opensource)
  - Has: 610 NIfTI segmentation masks (vertebrae C1~S1+sacrum)
  - Missing: Paired CT images (gt/ folder only has *_seg.nii.gz, no CT volumes)
  - To use: Need to download CT images via TCIA (scripts/download_ctspine1k_images.py exists)
  - Decision: SKIP for now — requires download, not immediately available

CANDIDATE B: OLIF dataset (0022_OLIF_dataset)
  - Has: 150k+ DICOM + 5021 labeled JPG + 2218 JSON (polygon)
  - Issue: Clinical data (lumbar surgery cases), potential PHI
  - Issue: OLIF-specific views (not standard anatomy atlas views)
  - Decision: SKIP — PHI concern, specialized surgical views

CANDIDATE C: LIDC Opensource (0013)
  - Has: Lung nodule CT patches in HDF5 format
  - Issue: Patch-based format (256x256), not full CT volumes
  - Issue: Nodule detection dataset, not anatomy atlas
  - Decision: SKIP — wrong use case

CANDIDATE D: SBJ_LLXR labeled X-ray (SELECTED)
  - Has: Labeled PNG/JPG spine X-rays with labelme polygon labels
  - Lateral views: T-L spine (T5~S1), clean polygon per vertebra
  - AP views: L-spine AP (T10~S1)
  - Format: labelme JSON (polygon shapes) — easy to convert to BodyAtlas contour format
  - Decision: USE — immediately available, labeled, clean quality

================================================================================
DATA FORMAT CONVERSION
================================================================================

Input (labelme polygon JSON):
  {
    "shapes": [
      {"label": "L1", "shape_type": "polygon", "points": [[x1,y1], [x2,y2], ...]}
    ],
    "imageWidth": 679, "imageHeight": 926
  }

Output (BodyAtlas contour JSON):
  [
    {"id": 19, "name": "L1", "contours": [[[x1,y1], [x2,y2], ...]]}
  ]

Conversion: scripts/gen_spine_xray_atlas.py
  - Reads labelme JSON for both views
  - Maps vertebra names to structure IDs
  - Rounds float coordinates to 1 decimal
  - Handles multiple contours per label (1:N polygon→contour list)
